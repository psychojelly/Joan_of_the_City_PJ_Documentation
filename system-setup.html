Setup · Path 1

Running a Local Test

The fastest way to see a movement working — a single machine, a single headset (or the in-editor simulator), no city required.

Open the project in Unity Hub

Add the cloned folder and open with Unity 6000.0.26f1. Let packages resolve on first launch.

Load the sandbox scene

Open Assets/Scenes/_Sandbox.unity — a desk-friendly scene that simulates one movement without leaving the building.

Play in the Editor (simulator)

Press Play with the XR Device Simulator active to walk the scene with mouse + keyboard. No hardware needed.

Or deploy to a single headset

Build & Run to your device, put on the glasses, and trigger the opening beat from the Show-Control panel.

Drive it from Show-Control

Use Jump to Beat to scrub through the movement and confirm phantoms, audio, and light all fire correctly.

🧪

The sandbox uses a fixed local anchor instead of a real-world scan, so you can iterate on staging and cues without on-site tracking. Treat its layout as indicative, not final.

⚠️

The Editor lies about XR frame cost. Always confirm a movement's performance on-device before calling it done.

Setup · Path 2

Larger System Set Up

The full performance configuration: multiple synchronized headsets, a real-world anchor map, an operator running Show-Control, and audio routed for the venue.

Topology

One operator station runs Show-Control and holds the master Conductor Clock. Each audience headset runs the client build and subscribes to clock + presence over the local network. A real-world anchor map (captured during the location scout) localizes every device to the same physical stage.

NodeRole
Operator StationMaster clock, Show-Control console, E-stop authority.
Audience HeadsetsClient build; render phantoms/audio against shared anchors.
Anchor MapPre-scanned spatial map of each Stage Volume; shared to all devices.
Local NetworkLow-latency link carrying clock + presence sync.

Bring-up sequence

Stage the location

Arrive on-site, confirm the Stage Volume matches the scanned anchor map, and re-scan if the environment has changed (new construction, signage, season).

Start the operator station

Launch Show-Control. Confirm the Conductor Clock is healthy and the master timeline loads the running order.

Localize each headset

Deploy the client build to every headset and run the localization routine until each device resolves the shared anchors. Verify alignment against a known physical reference point.

Verify presence sync

Confirm all headsets report into Presence Sync and see a consistent world state — phantoms appear in the same place for everyone.

Sound check

Walk the route; confirm spatial audio sources sit where they should and attenuate naturally with distance and occlusion.

Dry run the running order

Operator advances each movement; the team watches for tracking drift, dropped cues, and comfort issues before any audience arrives.

📍

Tracking budget: outdoor city environments are hostile to tracking — moving people, changing light, reflective glass. Keep critical phantoms anchored near stable, textured surfaces, and never place a key dramatic moment where tracking is known to drift.

🛟

Always rehearse the E-stop before doors open. Every operator must know how to fade all content to a safe neutral state instantly. See Cues → Editing & Cueing System.

Pre-show checklist

  • All headsets charged and localized to the shared anchor map.
  • Operator station on mains power with network confirmed stable.
  • Running order loaded and verified against the latest movement builds.
  • Audio levels checked at the real performance time of day.
  • E-stop tested; safe route and seated/limited-mobility paths confirmed.
