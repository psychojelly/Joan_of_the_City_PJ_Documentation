Assets

The asset pipeline

Assets flow from source tools into the Unity project through a consistent path. Everything heavy is tracked with Git LFS so the repo stays usable.

Author in source tools

Audio in the DAW, models in Blender/Maya, textures in the painter of choice. Keep editable source files in the team drive, not the repo.

Export to spec

Export to the formats below, named to convention. Bake what can be baked.

Drop into the right folder

Place under Assets/Audio, Assets/Phantoms, or Assets/Art as appropriate.

Commit via LFS

Binary assets must land on LFS-tracked paths. Confirm with git lfs status before pushing.

📦

Never commit a binary outside an LFS-tracked path. A raw model or stem in normal git bloats the repo permanently for everyone.

Assets

Naming conventions

Predictable names make cues, search, and reviews painless. Lowercase, hyphen-separated, scoped by movement.

# pattern:  <movement>_<category>_<name>[_<variant>]

II_phantom_judge.fbx
II_audio_aria-1.wav
II_tex_judge-robe_emissive.png
II_anim_judge-rise.anim
shared_phantom_chorus.fbx        # reused across movements
TokenValues
movementI, II, III, or shared
categoryphantom, audio, tex, anim, mat
nameDescriptive, hyphen-separated (judge-robe)
variantOptional (emissive, lod1, alt)
🔗

Asset names should match the ids used in cue sheets. A cue that spawns "judge" should resolve to II_phantom_judge with no guesswork.

Design

Visual language

A coherent world for Joan: candlelit golds, sacred violets, and the cold blue of judgement. Operatic — high contrast, theatrical light, restraint everywhere else. Every asset should serve this palette.

Typography

Display — Cormorant Garamond. A high-contrast serif for titles, libretto, and any sacred text. It carries the operatic, historical weight.

Interface — Inter. Neutral and legible for the rare diegetic labels and all operator/console tooling.

Mono — JetBrains Mono. Code, beat ids, anchor names, cue sheets.

Light & material principles

  • Light tells the story. A movement's emotion is set by its key light color before any phantom appears.
  • Subtractive design. The real city is busy — virtual content should be sparse, deliberate, and luminous against the dark.
  • Glow as voice. Phantoms emit soft light; emissive intensity follows their vocal line.
  • Readable in daylight. Assets must hold up in changing outdoor light — test at the real performance time.
