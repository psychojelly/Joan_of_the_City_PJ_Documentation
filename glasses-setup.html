Setup & Operation

Glasses Setup

Bringing a brand-new (or factory-reset) phone and pair of XREAL glasses into the fleet — from first power-on to a show build running in the glasses.

Setup · From scratch

Setting up from scratch

Do these in order — several later steps silently depend on earlier ones.

Turn on the phone and set a PIN — and document it

Complete the Android first-boot setup, set a screen-lock PIN, and record the PIN in the team's device sheet. An undocumented PIN on a fleet device is a bricked device on show night.

Download the necessary XREAL / Nebula software

The device should prompt automatically once it's on (and/or the first time glasses are connected). Accept the prompts and let everything install.

Plug in the glasses and confirm they work

Verify the glasses actually display, and update any firmware they prompt for now — not later, and never mid-show.

Turn on AirCast in the glasses app

Enable “Default AirCast”. This removes the auto-starting of the default XREAL startup app — the Nebula environment that uses the phone as a controller — so it stops grabbing the glasses every time they're plugged in.

Download the Glasses Control app (the SDK controller)

Get Glasses Control from the app store — it does not ship pre-installed. It's the SDK server / launcher: the app that lists and launches show builds.

Enable Developer Mode on the phone

Sideloading needs Developer options. Open Settings → About phone and tap Build number seven times until it says “You are now a developer.” Then in Settings → System → Developer options, turn on USB debugging (and “Install via USB” if present).

Load the APK of the build you want to use

Sideload the show build onto the phone. See the warning below — the build must be made for XREAL.

Plug into the glasses — the app should start

If Nebula starts instead: close Nebula, open the Glasses Control app, and restart the show app by hitting its Restart button.

⚠️

The build must include the XREAL plugin. Make sure the APK was built for XREAL (the XREAL loader enabled in Unity's XR Plug-in Management). A build without it installs and opens normally but has no head tracking and won't appear in the Glasses Control app — a failure mode that looks like broken hardware and is very easy to lose nights to.

Setup · Operating

Day-to-day launch

Once a device is provisioned, the reliable launch sequence is:

Charge the host

Make sure the phone is well charged — a low host under-feeds the glasses over USB-C and the link can drop under load.

Plug in the glasses and wait

Wait until the glasses render in 3D — that confirms the display has negotiated into stereo mode.

Launch (or restart) the show app

The app should start on its own; if Nebula appears, close it and use Restart in Glasses Control.

Leave the cable alone

Don't plug/unplug the glasses while the app runs. If anything glitches: quit the app → replug the glasses → wait for stereo → relaunch. Worst case, reboot the phone.
