System Overview

How the show is built: independent systems, coordinated by a single timeline, communicating through events.

Architecture

Architecture at a glance

Joan is built as a set of independent systems coordinated by one master timeline. Each system does exactly one thing and talks to the others only through events — so movements stay swappable and the show stays resilient if any single system stalls.

🧱

Rule of thumb: if you find yourself referencing one manager from inside another, raise an event instead. Movements should be assemblable and removable without touching engine code.

The Systems

Six systems make up the runtime. Everything else is content layered on top of them.

SystemResponsibility
Conductor ClockMaster timeline. Emits beat events that every other system listens to.
Anchor ServiceResolves and persists real-world anchors; keeps virtual content pinned to the city.
Phantom DirectorSpawns, animates, and retires virtual performers per beat.
Spatial AudioPositions voices & score in 3D; ducks and crossfades between movements.
Show-ControlOperator console: start, pause, jump-to-beat, emergency stop.
Presence SyncKeeps co-located audience members in a consistent shared scene.
Architecture

Communication: events, not references

Systems never call each other directly. The Conductor Clock raises beat events; subscribers react. This keeps movements swappable and makes the show fault-tolerant.

// One event bus. Every cue is a beat.
public static class ConductorClock {
    public static event Action<Beat> OnBeat;

    public static void Fire(Beat beat) => OnBeat?.Invoke(beat);
}

// The Phantom Director just listens.
ConductorClock.OnBeat += beat => {
    if (beat.id == "II.trial.enter")
        phantomDirector.Spawn("judge", anchor: "courthouse.dais");
};
Architecture

Runtime data flow

A single beat travels a predictable path from the operator's intent to the audience's senses.

Operator or trigger

The Show-Control console (or an audience-position trigger) advances the Conductor Clock.

Beat fires

The clock raises a Beat event onto the bus.

Systems react

Phantom Director, Spatial Audio, and lighting each respond to the beats they care about.

Anchored to the world

The Anchor Service positions all content against real-world anchors.

Synced across audience

Presence Sync ensures every headset sees the same world state.

Project layout

Where things live in the Unity project. Content (movements, audio, art) is kept separate from systems (engine code).

Assets/
├─ Scenes/          # _Sandbox + one scene per movement
├─ Systems/         # Clock, Anchors, Audio, ShowControl, Presence
├─ Phantoms/        # Performer prefabs, rigs, animation
├─ Movements/       # Per-movement config (beats, anchors, cues)
├─ Audio/           # Score stems + spatial SFX (LFS)
└─ Art/             # Materials, shaders, light rigs
