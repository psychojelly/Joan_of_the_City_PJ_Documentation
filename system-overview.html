Architecture

Architecture at a glance

The show is built in Unity as a set of concerns layered over each scene: anchoring holds content in the real world, cueing drives what happens and when, positioning keeps spawns consistent, and a performance budget keeps it all running on-headset. The whole thing must behave identically across every device in the room.

Architecture

Cueing & STOP ALL

Each scene advances through cues that trigger light, particles, ravens, shadow states, and audio. Two things matter above all: cues must fire the same way on every device, and the operator must always be able to halt safely.

Cues are fired over OSC from a web-based operator controller through a small proxy — see The OSC control system.

Cue consistency

  • The same cue must produce the same result on every headset in the room — divergence breaks the shared experience.
  • Known issue under active work: unpredictable cue behavior in Light and Voice.
  • Cue placement and timing are authored as data — see the Cues page for the cue-sheet model.

STOP ALL

STOP ALL is the master safety: it halts and resets running cues across the show. Its rewiring and functionality must be verified before every performance — never run an audience without confirming it.

🛟

Treat STOP ALL as a release blocker. If it isn't confirmed working on the current build, the show is not ready.

Open questions

Decisions still in flight. These block or shape downstream work and should be resolved with owners assigned.

  • Anchoring strategy — final approach to permanent anchoring; whether GUIDs solve persistence; the long-term image-marker decision.
  • Networking requirements — how multiple headsets stay in sync. Substantially addressed by the Audio Sync Design (shared clock, scheduled starts, unicast + redundancy); the remaining piece is the Unity-side implementation.
  • Device compatibility — target headset count and the brain-PC-per-headset arrangement for the XREAL glasses.
  • Performance budgets — agreed particle and texture budgets per scene.
  • Scene measurements — real-world dimensions per scene (captured during setup; see Larger System Set Up).
